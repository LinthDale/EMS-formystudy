## 2. 每個容器詳細說明

### 2-1. ems-simulator

**做什麼**
- 跑一個假的 Modbus TCP slave（port 5020），每秒更新 4 個暫存器：voltage、current、power_kw、energy_kwh
- 同時跑一個 FastAPI（port 8001），讓你在不 rebuild 的情況下調整模擬參數或注入故障

**檔案位置**：`services/simulator/`
- `src/main.py`：所有程式碼（Modbus slave + FastAPI 合跑在一個 asyncio process）
- `requirements.txt`：**鎖 `pymodbus==3.6.9`**，別升 3.13（API 大改過）
- `Dockerfile`

**有狀態嗎**：無。重啟就清零（energy_kwh 會歸零）

**關鍵指令**
```bash
# 看假電表現在的狀態
curl http://localhost:8001/health
curl http://localhost:8001/config

# 改模擬參數（不重啟容器）
curl -X POST "http://localhost:8001/config?power_base_kw=120"

# 注入故障測告警
curl -X POST "http://localhost:8001/inject-fault?mode=freeze"   # 值凍結
curl -X POST "http://localhost:8001/inject-fault?mode=zero"     # 值歸零
curl -X POST "http://localhost:8001/inject-fault?mode=none"     # 恢復正常

# 改了程式要重 build
docker compose up -d --build simulator
```

### 2-2. ems-gateway

**做什麼**
- Telegraf 每秒連 `simulator:5020` 讀 holding register
- 用 Modbus 新版 `request` 語法解碼 INT16 / FLOAT32 + scale
- 結果用 Influx Line Protocol 格式 publish 到 `ems/devices/sim-001/measurements`

**檔案位置**：`services/gateway/telegraf.conf`

**有狀態嗎**：無。重啟即刻恢復

**關鍵指令**
```bash
# 改 telegraf.conf 後（不用 rebuild、直接 restart 重讀 config）
docker compose restart gateway

# 看它在吐什麼
docker compose logs -f gateway

# 常見錯誤：連不到 simulator
#   → 檢查 simulator 是否 healthy、controller URL 是否對
# 常見錯誤：Modbus decode 值怪
#   → 確認 byte_order（ABCD/CDAB/BADC/DCBA）與 address 對
```

**未來擴充**：接真實電表時在同一份 conf 加 `[[inputs.modbus.request]]` 區塊，或另起一個 `gateway-xxx` 容器

**檔案位置**：`infra/mosquitto/mosquitto.conf`

```bash
  -t 'ems/devices/test/measurements' \
  -m 'electricity_measurements,device_id=test voltage=220 1776900000000000000'
```

- 加使用者密碼（`password_file`）
- 開 TLS
- 換 **EMQX**（Apache 2.0，功能更多）

**做什麼**
- Telegraf 訂閱 `ems/devices/+/measurements`（通配所有設備）

**檔案位置**：`services/ingest/telegraf.conf`

**有狀態嗎**：有 5 秒 flush buffer。若容器強殺可能丟那 5 秒

**關鍵指令**
```bash
# 改 conf

# 看寫入情況
docker compose logs -f ingest

# 常見錯誤：authentication failed
#   → .env 的 POSTGRES_PASSWORD 跟 DB 實際不一致
# 常見錯誤：relation "electricity_measurements" does not exist
#   → init.sql 沒跑起來或 rename migration 未執行。需要 docker compose down -v 重建（新部署）或跑 migrations/000_rename_measurements.sql（既有部署）
```

### 2-5. ems-timescaledb ⚠️ 最關鍵

**做什麼**
- PostgreSQL 15 + TimescaleDB 擴充
- 存放所有量測歷史、未來所有業務表

**檔案位置**
- `infra/timescaledb/init.sql`：schema（表、hypertable、api view、web_anon role）**只在第一次建容器時跑**
- `infra/timescaledb/02-authenticator.sh`：建 authenticator role（用 env var 的密碼）**同上**

**有狀態嗎**：**有！資料存在 Docker volume `timescale_data`**
- `docker compose stop / down` → volume 保留
- `docker compose down -v` → **資料全刪**

**關鍵指令**
```bash
# 連進去
docker exec -it ems-timescaledb psql -U postgres -d ems

# 用 TablePlus / DBeaver 連：host=127.0.0.1 port=5432 user=postgres db=ems

# 查最新資料
docker exec ems-timescaledb psql -U postgres -d ems -c \
  "SELECT time, voltage, power_kw FROM electricity_measurements ORDER BY time DESC LIMIT 5;"

# 查資料量
docker exec ems-timescaledb psql -U postgres -d ems -c \
  "SELECT COUNT(*), MIN(time), MAX(time) FROM electricity_measurements;"

# 備份整個 DB（建議定期跑）
docker exec ems-timescaledb pg_dump -U postgres ems > backup_$(date +%F).sql

# 還原
cat backup_2026-04-23.sql | docker exec -i ems-timescaledb psql -U postgres -d ems

# 清髒資料
docker exec ems-timescaledb psql -U postgres -d ems -c \
  "DELETE FROM electricity_measurements WHERE power_kw > 1000;"
```

**init.sql 改了怎麼重跑**

因為 init 只在 volume 為空時跑，改 schema 後要：
```bash
# 方法 A：備份 → 砍 volume → 重建 → 還原
docker exec ems-timescaledb pg_dump -U postgres ems > backup.sql
docker compose down -v
docker compose up -d
# 等 timescaledb healthy
cat backup.sql | docker exec -i ems-timescaledb psql -U postgres -d ems

# 方法 B：手動下 ALTER / CREATE 指令（推薦，production-safe）
docker exec -it ems-timescaledb psql -U postgres -d ems
# 在 psql 裡手動跑 ALTER TABLE / CREATE INDEX 等
```

### 2-6a. ems-grafana

**做什麼**
- 儀表板（2 個）：`ems-overview`（6 panels 即時電力）+ `ems-device-health`（5 panels，PRD-0004 device-service 健康度）
- 告警引擎（4 群組）：業務（功率）/ 管線健康（寫入停滯）/ **LLM 預算（PRD-0004 FR-400~405）** / 設備分類安全（FR-339/344）
- 通知：觸發時送 Telegram，恢復時送 RESOLVED

**檔案位置**：`infra/grafana/provisioning/`
- `datasources/timescaledb.yaml`：資料來源（接 TimescaleDB）
- `dashboards/ems-overview.json`：即時電力儀表板（6 panels）
- `dashboards/device-service-health.json`：**device-service 健康度**（5 panels：待確認佇列 / 狀態分布 / 預算 gauge / 成本明細 / 負面事件；uid `ems-device-health`）。由 `infra/grafana/_build_device_health.py` 生成，**勿手改 JSON**
- `alerting/contact-points.yaml`：Telegram bot token + chat_id（**chatid 在此 YAML 硬寫，env var 無效**）
- `alerting/rules.yaml`：告警規則（4 群組）。預算告警 `EMS-預算告警` 比例取 ledger `budget_usd`、0.8 硬編（改 `BUDGET_WARN_RATIO` env 須同步此檔）；收到預算告警的處置 SOP 見操作手冊 §3.7
- `alerting/notification-policies.yaml`：路由（全部送 Telegram）

**有狀態嗎**：plugin cache 在 volume `grafana_data`，儀表板 / 告警全在 provisioning 檔案，**重建容器不會遺失任何設定**

**關鍵指令**
```bash
# 打開儀表板（瀏覽器）
http://localhost:3000
# 登入：admin / admin（或 .env 的 GRAFANA_PASSWORD）
# 直達儀表板：http://localhost:3000/d/ems-overview（電力）｜ http://localhost:3000/d/ems-device-health（device-service 健康度）

# 改 provisioning 後立即生效（不用重啟容器）
curl -s -u admin:admin -X POST http://localhost:3000/api/admin/provisioning/alerting/reload
curl -s -u admin:admin -X POST http://localhost:3000/api/admin/provisioning/dashboards/reload
curl -s -u admin:admin -X POST http://localhost:3000/api/admin/provisioning/datasources/reload

# 測試 Telegram 通知（手動送一則）
curl -u admin:admin -X POST http://localhost:3000/api/alertmanager/grafana/config/api/v1/receivers/test \
  -H 'Content-Type: application/json' \
  -d '{"receivers":[{"name":"Telegram","grafana_managed_receiver_configs":[{"uid":"telegram-main","name":"Telegram","type":"telegram","settings":{"bottoken":"<your-token>","chatid":"7171144544","message":"EMS test"}}]}]}'

# 查 datasource 健康
curl -s -u admin:admin http://localhost:3000/api/datasources/uid/timescaledb-ems/health

# 查目前告警狀態
curl -s -u admin:admin http://localhost:3000/api/prometheus/grafana/api/v1/rules \
  | python3 -c 'import json,sys; gs=json.load(sys.stdin)["data"]["groups"]; [print(r["name"], r["state"]) for g in gs for r in g["rules"]]'

# 改了 docker-compose.yml 的 grafana 區塊後
docker compose up -d grafana   # 會 recreate（不要用 docker restart）
```

**重要已知行為**
- `docker restart ems-grafana` 會讓容器離開 `ems_default` 網路，**永遠用 `docker compose up -d grafana`**
- `chatid` 純數字型別問題：Grafana provisioning 把數字 env var 解成 JSON number，改 chat_id 請直接編輯 `contact-points.yaml` 的 `"7171144544"` 那行
- 首次 `docker compose up -d` 時 Grafana 啟動比 timescaledb 快，會有一次連線失敗紀錄，屬正常

### 2-12. ems-query

**做什麼**

- 目前只有 `GET /electricity_measurements` 可用（read-only via `web_anon` role）

### 2-13. ems-device-service（PRD-0003 設備 Registry + 自動分類 REST API）

- **Port 8002**。三組 X-API-Key 通道（FR-310）：OPS（CRUD/confirm/override/reject/ai-feedback）、INGEST、AI。
- 雙 asyncpg pool（ADR-017）：`device_service_ai`（分類路徑，無 measurements SELECT）/ `device_service_ops`（CRUD、讀 measurements 取樣本）。
- 凍結 trigger（migration 010/011）：human/manual_override/migration_backfill 的紀錄非經 freeze-override token 不可改。
- audit 落 `device_audit_log`（append-only；FR-339/FR-344 告警走 Grafana SQL window query）。
- **無狀態**，可重啟；schema 由 `infra/timescaledb/migrations/003-015` 套用（手動套到 dev DB；非容器初始化）。

### 2-14. ems-device-service-mcp（device-service MCP 端點，PRD-0003 §8.2 / ADR-012）

- 與 `ems-device-service` **同一 image，不同 command**（跑 `uvicorn device_service.mcp_server:app`）。Streamable HTTP，`127.0.0.1:8766`（只綁本機 host map；container 內 0.0.0.0 才能讓 host map 連到）。
- **AI 通道三個 tool**：`list_low_confidence_candidates` / `get_device_digest` / `classify_with_context`。**無** confirm/override/reject（§8.6.7：那些只走 OPS REST）。
- 認證：`X-API-Key: $AI_API_KEY`（timing-safe 比對；空 key fail-closed 全拒）。`/healthz` 免認證（容器 healthcheck 用）。
```bash
# 健康檢查 / 認證驗證
curl http://127.0.0.1:8766/healthz                       # -> {"status":"ok"}
curl -X POST http://127.0.0.1:8766/mcp                    # 無 key -> 401
```
- **⚠️ AI_API_KEY rotation：key 在 `build_app()` 啟動時讀一次 → rotate 後必須 `docker compose up -d device-service-mcp`（或 restart）才生效。**
- **⚠️ rate limit 現況（single-AI-client 假設）**：per-IP 60/min，in-memory per-process。經 docker-proxy host map 後所有來源共用 bridge gateway IP → 實質是 **global 60/min**。多 AI client 會互相卡；未來要 API-key-scoped 或 Redis-backed limiter（待 ADR）。
- 每次 tool call 寫結構化 audit log（`tool_name`/`caller_ip`/`device_id`/`result_status`/`latency_ms`）；ToolError 只回穩定錯誤碼/安全訊息，內部/DB 細節只進 server log。

## 3. 日常操作速查

### 3-1. 啟停場景

| 想做的事 | 指令 | 資料保留？ |
|---------|------|----------|
| 暫停幾分鐘（電力等）| `docker compose pause` / `unpause` | ✅ |
| 暫停幾小時~幾天 | `docker compose stop` / `start` | ✅ |
| 關機 / 重開機 | `docker compose down` / `up -d` | ✅（volume 保留）|
| 改 config 要重建容器 | `docker compose up -d --build <service>` | ✅ |
| 砍掉重練（**資料全刪**）| `docker compose down -v` / `up -d` | ❌ |
| 只重啟某個 | `docker compose restart <service>` | ✅ |

### 3-2. 看日誌

```bash
docker compose logs                        # 全部服務，一次看
docker compose logs -f                     # 持續追蹤全部
docker compose logs -f --tail=50 gateway   # 只看 gateway 最後 50 行然後追
docker compose logs --since=10m ingest     # ingest 近 10 分鐘
```

### 3-3. 進容器 shell

```bash
# Python 容器（simulator 有 bash）
docker exec -it ems-simulator bash

# Alpine 底的（mosquitto、postgrest）只有 sh
```

### 3-6. 資料庫備份還原

```bash
# 備份（存到當前目錄）
docker exec ems-timescaledb pg_dump -U postgres ems > ems_backup_$(date +%F).sql

# 備份壓縮
docker exec ems-timescaledb pg_dump -U postgres ems | gzip > ems_backup_$(date +%F).sql.gz

# 還原（假設 DB 已清空）
cat ems_backup_2026-04-23.sql | docker exec -i ems-timescaledb psql -U postgres -d ems

# 從 gz 還原
gunzip -c ems_backup_2026-04-23.sql.gz | docker exec -i ems-timescaledb psql -U postgres -d ems
```

## 4. 常見故障排除決策樹

```
docker compose ps
├── 某容器 = "Restarting" / "Exited"
│   └──▶ docker compose logs --tail=50 <service>
│         ├── 看錯誤訊息
│         │   ├── 設定錯   → 改 config 檔 → restart
│         │   ├── 相依沒起 → 等 / 看 depends_on 的服務
│         │   └── 權限問題 → 檢查 .env 的密碼是否跟 DB 實際一致
│
├── 全部 = "Up" 但 API 沒回應
│   └──▶ 看 query logs（最常是 PostgREST 連 DB 失敗）
│         ├── "could not translate host name" → 密碼含特殊字元、URI 解析壞
│         ├── "password authentication failed" → .env 跟 DB 不同步
│         └── "schema cache" 錯誤 → docker compose restart query
│
├── DB 有資料但 REST 回 []
│   └──▶ 確認 api schema 的 view 存在
│         docker exec ems-timescaledb psql -U postgres -d ems \
│           -c "\dv api.*"
│
└── 日期/時間怪
    └──▶ TimescaleDB 預設 UTC，TablePlus 顯示可能加減時區
```

## 5. 安全基線（目前是 dev 設定，上 production 前必改）

| 項目 | 現在 | Production |
|------|------|-----------|
| Mosquitto auth | anonymous | 使用者名+密碼 / mTLS |
| MCP server | 127.0.0.1 only, no auth | reverse proxy + auth / VPN / SSH tunnel |
| KC Modbus simulator | 5021 對外 | production 移除或只放內網測試段 |
| PostgreSQL port | 5432 對外 | 只對 Docker network，不 publish |
| PostgREST | HTTP | HTTPS（前面加 nginx）；demo 不直接公開 PostgREST |
| Grafana 對外 demo | localhost / dev port | 只經 Cloudflare Tunnel + Access 公開讀取；見 `doc/operations/network/Cloudflare_Grafana_Demo_對外公開操作指南.md` |
| `.env` 密碼 | 弱 | 強 random，放 secret manager |
| Mosquitto 持久化 | false | true + 加磁碟 volume |
| DB 備份 | 手動 | cron 自動、異地備份 |
